## Project Origins

**pdf2esx** was born from a real-world need in the roofing industry. Contractors were spending hours manually transcribing insurance scope documents into their own estimate formats—a tedious, error-prone process that delayed claims and cost money.

### The Vision

> "Could have that shit working by Friday" - Project Kickoff Meeting, January 2026

This project aims to transform a 2+ hour manual process into a 5-minute automated workflow, helping contractors:

- **Save Time**: Automated data extraction eliminates manual entry
- **Recover More Value**: Proper supplementation recovers $15K-$25K+ per claim
- **Reduce Errors**: OCR and validation ensure accurate data transfer
- **Scale Operations**: Process more claims with fewer resources

## Technical Approach

### Why Node.js + TypeScript?

After evaluating multiple approaches, we chose Node.js with TypeScript for several reasons:

1. **Puppeteer Integration**: Best-in-class HTML to PDF conversion
2. **Async Excellence**: Native handling of API calls and file operations
3. **Type Safety**: TypeScript catches errors before runtime
4. **NPM Ecosystem**: Rich library support for PDF, OCR, and API integrations
5. **PRSMTECH Stack Consistency**: Aligns with our Next.js projects

### Why HTML Templates Over SVG?

While SVG was initially considered for "best quality" output, HTML/CSS templates offer:

- **Dynamic Content**: Easier handling of variable-length line item tables
- **Maintainability**: Standard web technologies, easier to customize
- **Browser Preview**: Live development with instant visual feedback
- **Puppeteer Rendering**: Excellent PDF output quality
